# Selenium Automation Framework

A complete, production-ready Selenium automation framework built with Java, TestNG, and Maven. This framework is designed for testing web applications with a focus on clean code, maintainability, and scalability.

## Features

- ✅ **Page Object Model (POM)**: Organized page classes for better maintainability
- ✅ **TestNG Integration**: Comprehensive test framework with data-driven testing
- ✅ **Cross-Browser Testing**: Support for Chrome and Edge browsers
- ✅ **WebDriverManager**: Automatic WebDriver management
- ✅ **Data-Driven Testing**: Apache POI for Excel-based test data
- ✅ **Extent Reports**: Beautiful HTML reports with screenshots
- ✅ **Screenshot Capture**: Automatic screenshots on test failure
- ✅ **Maven Build**: Complete Maven configuration for CI/CD integration
- ✅ **Logging**: Comprehensive console and file logging

## Prerequisites

- Java 11 or higher
- Maven 3.6.0 or higher
- Git (optional, for version control)

## Installation

1. **Clone/Download the project**
   ```bash
   git clone <repository-url>
   cd AutomationFramework
   ```

2. **Install Maven dependencies**
   ```bash
   mvn clean install
   ```

3. **Create test data Excel file** (Run once)
   ```bash
   mvn exec:java -Dexec.mainClass="utils.ExcelDataCreator"
   ```

## Configuration

### Browser Selection

Edit `testng.xml` to change the browser parameter:

```xml
<parameter name="browser" value="chrome"/>  <!-- Use 'chrome' or 'edge' -->
```

### Base URL

Edit `BaseTest.java` to change the base URL:

```java
protected String baseURL = "https://www.saucedemo.com/";
```

### Test Data

- Test data is stored in `test-data/LoginTestData.xlsx`
- Add/modify test cases data in Excel as needed
- Excel file has sheets with headers and data rows

## Running Tests

### Run All Tests
```bash
mvn clean test
```

### Run Specific Test Class
```bash
mvn test -Dtest=LoginTest
```

### Run Specific Test Method
```bash
mvn test -Dtest=LoginTest#testValidLogin
```

### Run with Specific TestNG XML
```bash
mvn test -DsuiteXmlFile=testng.xml
```

### Run Tests in Parallel
Edit `testng.xml` and change `thread-count`:
```xml
<suite name="Selenium Automation Framework Suite" thread-count="5">
```

## Test Cases

### LoginTest
- `testValidLogin()` - Tests successful login with valid credentials
- `testInvalidLogin()` - Data-driven test for invalid logins (reads from Excel)
- `testLockedUserLogin()` - Tests locked user scenario
- `testEmptyUsernameLogin()` - Tests login with empty username
- `testEmptyPasswordLogin()` - Tests login with empty password

### CartTest
- `testAddProductToCart()` - Tests adding single product to cart
- `testAddMultipleProductsToCart()` - Tests adding multiple products
- `testViewCart()` - Tests viewing cart contents
- `testRemoveProductFromCartOnHomePage()` - Tests removal from home page
- `testRemoveProductFromCartPage()` - Tests removal from cart page
- `testRemoveAllProductsFromCart()` - Tests removing all items
- `testVerifySpecificItemInCart()` - Tests item verification
- `testCheckoutButtonVisibility()` - Tests checkout button visibility

### LogoutTest
- `testLogout()` - Tests logout functionality
- `testLogoutAfterAddingToCart()` - Tests logout after adding items
- `testLoginAfterLogout()` - Tests re-login after logout
- `testCartClearedAfterLogout()` - Tests cart behavior after logout

## Page Object Model Classes

### LoginPage
Methods:
- `login(String username, String password)` - Performs login
- `getErrorMessage()` - Gets error message
- `isLoginPageLoaded()` - Checks if login page is loaded
- `isErrorMessageDisplayed()` - Checks if error is displayed
- `getPageTitle()` - Gets page title
- `getCurrentUrl()` - Gets current URL

### HomePage
Methods:
- `isHomePageLoaded()` - Checks if home page is loaded
- `isUserLoggedIn()` - Checks if user is logged in
- `addProductToCart(int index)` - Adds product by index
- `addProductToCartByName(String productName)` - Adds product by name
- `removeProductFromCart(int index)` - Removes product by index
- `getCartItemCount()` - Gets number of items in cart
- `clickOnCart()` - Navigates to cart
- `openBurgerMenu()` - Opens menu
- `logout()` - Performs logout

### CartPage
Methods:
- `isCartPageLoaded()` - Checks if cart page is loaded
- `getCartItemCount()` - Gets count of items
- `getCartItemNames()` - Gets list of item names
- `getCartItemPrices()` - Gets list of item prices
- `removeItemByIndex(int index)` - Removes item by index
- `removeAllItems()` - Removes all items
- `isCartEmpty()` - Checks if cart is empty
- `isItemInCart(String itemName)` - Checks if item exists
- `clickCheckout()` - Clicks checkout
- `isCheckoutButtonVisible()` - Checks checkout button visibility

## Utility Classes

### ExcelReader
Reads test data from Excel files:
- `getCellData(int row, int column)` - Get cell value
- `getCellDataByHeader(int row, String header)` - Get by header
- `getRowDataAsMap(int row)` - Get entire row as Map
- `getAllRowsAsMapList()` - Get all rows as List

### ScreenshotUtil
Captures screenshots:
- `takeScreenshot(WebDriver driver, String testName)` - Capture with auto naming
- `takeScreenshotWithCustomName(WebDriver driver, String fileName)` - Custom name
- `getScreenshotDirectory()` - Get screenshots directory path
- `clearScreenshotDirectory()` - Clear all screenshots

### ExcelDataCreator
Generates test data Excel file:
- `createLoginTestData()` - Creates LoginTestData.xlsx

## Reports

### Extent Reports
- Generated after test execution
- Location: `reports/ExtentReport_<timestamp>.html`
- Features:
  - Test summary dashboard
  - Individual test details
  - Screenshots on failure
  - System information

### Screenshots
- Automatically captured on test failure
- Location: `screenshots/` directory
- Naming: `<testname>_<timestamp>.png`
- Attached to Extent Report

## Best Practices Implemented

1. **Page Object Model**: Separates test logic from page elements
2. **Base Test Class**: Centralized WebDriver setup/teardown
3. **Reusable Methods**: Common operations in utility classes
4. **Data-Driven Testing**: Uses external data sources (Excel)
5. **Assertions**: TestNG assertions for validation
6. **Waits**: Proper WebDriverWait for element synchronization
7. **Logging**: Console output for debugging
8. **Exception Handling**: Proper error handling and reporting
9. **Naming Convention**: Clear, descriptive names for classes and methods
10. **Documentation**: Comprehensive JavaDoc comments

## Troubleshooting

### WebDriver Issues
- Ensure WebDriverManager is properly configured
- Check internet connection (WebDriverManager downloads drivers)
- Verify browser is installed (Chrome/Edge)

### Excel File Issues
- Ensure `LoginTestData.xlsx` exists in `test-data/` folder
- Run `ExcelDataCreator` if file is missing
- Verify Excel file structure matches expected format

### Test Failures
- Check screenshots in `screenshots/` folder
- Review Extent Report for detailed failure information
- Check console output for debug information
- Verify element locators are correct for current website version

## CI/CD Integration

### GitHub Actions Example
```yaml
- name: Run Tests
  run: mvn clean test
  
- name: Publish Reports
  uses: actions/upload-artifact@v2
  with:
    name: Test Reports
    path: reports/
```

### Jenkins Example
```groovy
stage('Test') {
    steps {
        sh 'mvn clean test'
    }
}

stage('Reports') {
    steps {
        publishHTML([
            reportDir: 'reports',
            reportFiles: 'index.html',
            reportName: 'Test Report'
        ])
    }
}
```
